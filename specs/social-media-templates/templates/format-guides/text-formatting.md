## 概述

文本格式规范旨在统一朋友圈内容的呈现方式，提升阅读体验，确保信息传达的有效性。规范涵盖字体、段落、标点、空格、换行等方面。

## 基本格式规范

### 1. 字体使用规范

#### 1.1 字体选择
- **主要字体**：使用系统默认字体，确保兼容性
- **特殊字符**：谨慎使用特殊字体或字符
- **表情符号**：使用通用表情符号，确保显示正常

#### 1.2 字体样式
- **加粗**：`**加粗文本**` 用于强调关键信息
- *斜体*：`*斜体文本*` 用于引用或特殊说明
- ~~删除线~~：`~~删除线文本~~` 用于表示已取消或更正
- `代码块`：`` `代码文本` `` 用于技术术语或代码

**使用示例**：
```markdown
**重要提示**：本次活动*仅限*前50名报名者。
原价~~199元~~，现价99元！
需要安装 `Python 3.8+` 版本。
```

### 2. 段落规范

#### 2.1 段落长度
- **理想长度**：每段2-4行（移动端显示）
- **最大长度**：不超过6行
- **短段落**：重要信息可以单独成段

#### 2.2 段落间距
- **段间距**：空一行分隔段落
- **行间距**：使用系统默认行间距
- **视觉分组**：相关内容可以适当靠近

#### 2.3 段落结构
1. **开头段**：吸引注意力，引出主题
2. **主体段**：详细内容，逻辑清晰
3. **结尾段**：总结或引导互动

### 3. 标点符号规范

#### 3.1 中文标点
- **全角标点**：使用全角标点符号
- **句号**：使用"。"而不是"."
- **逗号**：使用"，"而不是","
- **引号**：使用「」或“”而不是""

#### 3.2 英文标点
- **半角标点**：英文内容使用半角标点
- **空格规则**：英文标点后加空格
- **专有名词**：保持原格式，如iPhone、YouTube

#### 3.3 特殊标点
- **省略号**：使用"……"而不是"..."
- **破折号**：使用"——"而不是"--"
- **连接号**：使用"-"连接数字或英文单词

### 4. 空格使用规范

#### 4.1 中文空格
- **全角空格**：中文之间使用全角空格
- **标点前后**：标点前后不加空格
- **数字英文**：中文与数字、英文间加空格

#### 4.2 英文空格
- **单词间**：英文单词间使用半角空格
- **标点后**：英文标点后加半角空格
- **特殊格式**：保持原格式的空格规则

#### 4.3 示例对比

**正确示例**：
```
今天去了北京故宫，参观了太和殿。
iPhone 14 Pro 的拍照效果很棒。
活动时间：2023年12月15日 14:00-16:00
```

**错误示例**：
```
今天去了北京故宫,参观了太和殿。
iPhone14Pro的拍照效果很棒。
活动时间：2023年12月15日14:00-16:00
```

### 5. 换行与对齐

#### 5.1 换行规则
- **自然换行**：在语义完整处换行
- **避免断词**：不在单词中间换行
- **移动端适配**：考虑移动端显示效果

#### 5.2 对齐方式
- **左对齐**：默认使用左对齐
- **居中对齐**：特殊内容可以居中对齐
- **右对齐**：时间、签名等可以使用右对齐

#### 5.3 列表格式
- **无序列表**：使用`-`或`*`开头
- **有序列表**：使用`1.`、`2.`等
- **列表缩进**：保持一致的缩进

## 高级格式技巧

### 1. 标题层次

#### 1.1 标题级别
- **一级标题**：`# 标题`（慎用，可能过大）
- **二级标题**：`## 标题`（用于主要部分）
- **三级标题**：`### 标题`（用于子部分）

#### 1.2 标题使用
- **数量控制**：一篇内容不超过3个标题
- **语义相关**：标题要准确反映内容
- **视觉平衡**：标题与正文比例适当

### 2. 引用与注释

#### 2.1 引用格式
```
> 这是引用内容
> 可以多行引用
```

#### 2.2 注释说明
- **HTML注释**：`<!-- 注释内容 -->`（仅作者可见）
- **括号注释**：`（注：说明内容）`
- **脚注说明**：在文末统一说明

### 3. 特殊内容格式

#### 3.1 联系方式
```
📞 电话：138-xxxx-xxxx
📧 邮箱：example@email.com
📍 地址：XX市XX区XX路XX号
```

#### 3.2 时间日期
```
⏰ 时间：2023年12月15日 周五 14:00
📅 日期：12月15日-12月20日
🕒 时长：2小时
```

#### 3.3 价格信息
```
💰 价格：99元/人
🎫 门票：早鸟票 79元（12月10日前）
💳 支付：微信/支付宝/银行卡
```

## 表情符号使用规范

### 1. 表情符号选择

#### 1.1 通用表情
- **开心**：😊 😄 🎉
- **感谢**：🙏 ❤️ 💝
- **推荐**：👍 👌 🌟
- **疑问**：🤔 ❓ 💭

#### 1.2 场景表情
- **美食**：🍔 🍕 🍣 ☕
- **旅行**：✈️ 🏖️ 🗺️ 🏔️
- **学习**：📚 🎓 💡 🧠
- **运动**：🏃‍♂️ ⚽ 🏀 🏊‍♀️

### 2. 表情符号使用规则

#### 2.1 使用位置
- **开头**：1-2个，引出主题
- **重点**：关键信息后，强调作用
- **结尾**：1-2个，增强互动

#### 2.2 使用数量
- **适度原则**：每段不超过3个表情
- **避免重复**：不连续使用相同表情
- **语义相关**：表情与内容相关

#### 2.3 使用示例

**适当使用**：
```
周末去爬山了！🏔️
风景真的太美了！😍
推荐大家也去体验！👍
```

**过度使用**：
```
周末去爬山了！🏔️😄🌟
风景真的太美了！😍😊🎉
推荐大家也去体验！👍👌💯
```

## 话题标签规范

### 1. 标签格式

#### 1.1 基本格式
- **#标签内容**：使用`#`开头
- **无空格**：标签内不使用空格
- **中文优先**：优先使用中文标签

#### 1.2 标签类型
- **主题标签**：`#旅行日记` `#美食探店`
- **地点标签**：`#北京` `#上海外滩`
- **活动标签**：`#周末活动` `#展览参观`
- **心情标签**：`#开心每一天` `#小确幸`

### 2. 标签使用规则

#### 2.1 数量控制
- **推荐数量**：3-5个标签
- **最多数量**：不超过8个标签
- **最少数量**：至少1个相关标签

#### 2.2 内容相关
- **紧密相关**：标签与内容高度相关
- **避免无关**：不使用无关标签吸引流量
- **精准描述**：使用准确描述内容的标签

#### 2.3 排列顺序
1. **主题标签**：最主要的内容主题
2. **地点标签**：相关地点信息
3. **活动标签**：具体活动类型
4. **通用标签**：`#朋友圈` `#日常`

## 常见问题与解决方案

### 问题1：段落过长
**解决方案**：
1. 在语义完整处拆分段落
2. 使用列表格式整理信息
3. 添加小标题分隔内容

### 问题2：标点混乱
**解决方案**：
1. 统一使用中文全角标点
2. 英文内容使用半角标点
3. 使用标点检查工具

### 问题3：表情过多
**解决方案**：
1. 删除重复的表情符号
2. 保留关键位置的表情
3. 确保表情与内容相关

### 问题4：标签无关
**解决方案**：
1. 删除无关的话题标签
2. 添加准确描述内容的标签
3. 控制标签数量在合理范围
