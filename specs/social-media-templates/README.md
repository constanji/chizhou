# 朋友圈生成规范模板系统

**创建日期**：2025-12-05  
**版本**：1.0.0  
**用途**：用于生成朋友圈内容的规范模板系统

## 概述

本模板系统提供了一套完整的朋友圈内容生成规范，包含多种场景的模板、变量替换机制、格式规范、内容建议等。系统设计遵循以下原则：

1. **标准化**：统一的朋友圈内容格式和结构
2. **场景化**：针对不同场景提供专用模板
3. **可定制**：通过变量替换机制实现个性化
4. **易用性**：模板简单易懂，易于使用

## 核心组件

### 1. 朋友圈内容模板 (social-post-template.md)
基础模板，定义了朋友圈的标准结构和必需/可选元素。

### 2. 场景模板 (scenario-templates/)
针对不同使用场景的专用模板：
- **旅行场景**：适合旅行分享
- **美食场景**：适合美食分享
- **活动场景**：适合活动参与
- **成就场景**：适合成就展示
- **日常生活**：适合日常分享

### 3. 格式规范 (format-guides/)
内容格式的详细规范：
- **文本格式**：字体、段落、表情符号使用规范
- **图片指南**：图片选择、编辑、发布规范
- **话题标签**：话题标签的选择和使用规则

### 4. 变量系统 (variable-system/)
变量替换机制的详细说明：
- **变量参考**：所有可用变量的定义和用法
- **变量示例**：变量使用的实际示例

### 5. 命令模板 (commands/)
生成朋友圈内容的命令模板，指导如何根据模板生成具体内容。

### 6. 脚本工具 (scripts/)
辅助脚本，用于批量处理占位符替换等任务。

## 使用方法

### 基本使用流程

1. **选择模板**：根据内容类型选择合适的场景模板
2. **填写变量**：根据变量参考手册填写具体内容
3. **应用格式**：按照格式规范调整内容格式
4. **生成内容**：使用命令模板生成最终朋友圈内容
5. **发布前检查**：按照检查清单进行最终检查

### 变量替换示例

```markdown
# 使用变量替换前
今天去了[地点]，吃了[美食名称]，感觉[心情形容词]！

# 使用变量替换后
今天去了北京故宫，吃了北京烤鸭，感觉非常满足！
```

## 占位符系统

系统使用以下占位符格式：

### 动态占位符（运行时替换）
- `[日期]`：当前日期
- `[时间]`：当前时间
- `[地点]`：地点信息
- `[人物]`：人物名称

### 静态占位符（模板填充时替换）
- `[美食名称]`：具体的美食名称
- `[活动名称]`：具体的活动名称
- `[心情形容词]`：描述心情的形容词
- `[成就描述]`：成就的具体描述

### 格式占位符
- `[图片描述]`：图片的详细描述
- `[话题标签]`：相关的话题标签
- `[表情符号]`：合适的表情符号

## 最佳实践

### 内容创作
1. **真实性**：分享真实的生活体验
2. **价值性**：提供对他人有价值的信息
3. **互动性**：鼓励朋友互动和评论
4. **适度性**：保持适度的发布频率

### 格式规范
1. **段落清晰**：合理分段，便于阅读
2. **图文匹配**：图片与文字内容相关
3. **标签恰当**：使用相关且适量的标签
4. **时间合适**：选择合适的时间发布

### 隐私保护
1. **位置保护**：谨慎分享具体位置
2. **个人信息**：避免泄露敏感信息
3. **他人隐私**：尊重他人隐私权
4. **内容审查**：发布前审查内容安全性
