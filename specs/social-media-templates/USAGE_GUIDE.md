# 朋友圈模板系统使用指南

## 快速开始

### 方式一：通过智能体命令使用（推荐）

如果你使用的是集成了 speckit 工具的智能体系统，可以直接使用命令：

```
/speckit.generate_templates 我想生成一条关于今天去故宫的朋友圈
```

或者更具体地：

```
我想分享今天去故宫的经历，和朋友一起，天气很好，拍了很多照片，感觉很震撼
```

智能体会：
1. 自动识别这是旅行场景
2. 选择旅行场景模板
3. 提取关键信息（地点、人物、天气、感受）
4. 替换所有变量占位符
5. 生成完整的朋友圈内容

### 方式二：手动使用模板

如果你想手动使用模板，可以按照以下步骤：

## 详细使用流程

### 第一步：选择模板

根据你要分享的内容类型，选择合适的场景模板：

| 场景类型 | 模板文件 | 适用情况 |
|---------|---------|---------|
| 美食分享 | `templates/scenario-templates/food-post.md` | 餐厅、美食、烹饪 |
| 旅行分享 | `templates/scenario-templates/travel-post.md` | 旅行、景点、出游 |
| 活动参与 | `templates/scenario-templates/event-post.md` | 聚会、活动、会议 |
| 成就展示 | `templates/scenario-templates/achievement-post.md` | 完成项目、获得奖项 |
| 日常生活 | `templates/scenario-templates/daily-life-post.md` | 日常琐事、心情分享 |
| 通用内容 | `templates/social-post-template.md` | 其他类型内容 |

### 第二步：阅读模板结构

打开选中的模板文件，了解模板的结构：

1. **内容结构**：模板定义了哪些必需部分和可选部分
2. **占位符**：模板中使用了哪些变量占位符（如 `[地点]`、`[人物]`）
3. **格式要求**：每个部分的格式规范是什么

### 第三步：准备内容信息

根据模板要求，准备以下信息：

#### 基本信息
- **时间**：什么时候（今天、昨天、具体日期）
- **地点**：在哪里（具体地点名称）
- **人物**：和谁一起（朋友、家人、同事）
- **事件**：发生了什么（具体活动描述）

#### 情感信息
- **心情**：感受如何（开心、满足、震撼等）
- **评价**：如何评价（推荐、一般、不推荐）
- **体验**：具体体验描述

#### 实用信息（可选）
- **价格**：人均消费、费用
- **时间**：活动时长、营业时间
- **交通**：如何到达
- **注意事项**：需要知道的信息

### 第四步：填写模板

打开模板文件，按照以下步骤填写：

1. **替换标题占位符**：
   ```markdown
   # 美食场景朋友圈模板：[美食主题]
   ```
   替换为：
   ```markdown
   # 美食场景朋友圈模板：北京烤鸭
   ```

2. **填写各个部分**：
   根据模板的结构，逐个填写每个部分的内容。

3. **替换变量占位符**：
   ```markdown
   今天去了[地点]，吃了[美食名称]，感觉[心情形容词]！
   ```
   替换为：
   ```markdown
   今天去了全聚德，吃了北京烤鸭，感觉非常满足！
   ```

### 第五步：应用格式规范

填写完成后，参考格式规范文件调整内容：

1. **文本格式**（`templates/format-guides/text-formatting.md`）：
   - 检查段落长度（每段2-4行）
   - 检查标点符号使用
   - 检查表情符号使用

2. **图片指南**（`templates/format-guides/image-guidelines.md`）：
   - 确定需要配几张图片
   - 确定图片内容和顺序
   - 检查图片与文字匹配度

3. **话题标签**（`templates/format-guides/hashtag-rules.md`）：
   - 选择3-5个相关话题标签
   - 按重要性排列
   - 检查标签格式

### 第六步：最终检查

发布前，进行以下检查：

- [ ] 所有占位符都已替换
- [ ] 内容完整，逻辑连贯
- [ ] 格式规范，符合要求
- [ ] 表情符号使用恰当
- [ ] 话题标签相关且适量
- [ ] 没有敏感信息泄露
- [ ] 语句通顺，无错别字

## 实际使用示例

### 示例1：美食分享

**用户输入**：
```
今天和朋友去了一家新开的日料店，点了寿司和拉面，味道很不错，环境也很好，人均200左右
```

**使用流程**：
1. 选择模板：`food-post.md`（美食场景）
2. 提取信息：
   - 地点：新开的日料店
   - 人物：朋友
   - 美食：寿司、拉面
   - 评价：味道不错、环境好
   - 价格：人均200左右
3. 填写模板并替换变量
4. 应用格式规范
5. 生成最终内容

**生成结果示例**：
```
🍣 今天和朋友打卡了一家新开的日料店！

终于吃到了心心念念的寿司和拉面，味道真的很不错！寿司新鲜，拉面汤底浓郁，环境也很日式，氛围感满满。

人均200左右，性价比还可以，推荐给喜欢日料的朋友们！

#日料探店 #美食分享 #周末好去处
```

## 常见问题

### Q1: 如何选择合适的模板？

**A**: 根据你要分享的内容类型选择：
- 美食相关内容 → 美食模板
- 旅行、景点 → 旅行模板
- 聚会、活动 → 活动模板
- 成就、完成项目 → 成就模板
- 日常琐事 → 日常模板
- 不确定 → 基础模板

### Q2: 变量替换后语句不通顺怎么办？

**A**: 
1. 检查变量值是否准确
2. 调整句子结构
3. 添加连接词
4. 参考变量示例手册

### Q3: 可以同时使用多个模板吗？

**A**: 可以。如果内容涉及多个场景，可以：
1. 选择一个主要模板
2. 从其他模板中借鉴合适的部分
3. 组合成完整内容

### Q4: 如何自定义模板？

**A**: 
1. 复制现有模板文件
2. 根据需求修改结构
3. 调整变量定义
4. 保存为新模板

### Q5: 生成的內容可以直接发布吗？

**A**: 建议：
1. 检查所有占位符是否已替换
2. 检查格式是否符合规范
3. 检查内容是否真实可信
4. 检查是否有敏感信息
5. 确认无误后再发布

## 高级用法

### 批量生成

如果有多个相似内容需要生成，可以：
1. 准备一个信息列表（CSV或JSON格式）
2. 使用脚本工具（`scripts/placeholder-processor.sh`）批量处理
3. 生成多个朋友圈内容

### 模板组合

对于复杂内容，可以：
1. 选择主要场景模板
2. 从其他模板借鉴部分结构
3. 组合成完整内容

### 个性化定制

根据个人风格，可以：
1. 修改模板的语气风格
2. 调整表情符号使用习惯
3. 自定义话题标签偏好

## 最佳实践

1. **真实性**：分享真实的生活体验，不要编造
2. **价值性**：提供对他人有价值的信息
3. **互动性**：鼓励朋友互动和评论
4. **适度性**：保持适度的发布频率
5. **隐私保护**：谨慎分享具体位置和个人信息
