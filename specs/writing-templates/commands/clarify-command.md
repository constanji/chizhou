# 需求澄清命令模板

## 命令说明

`/writing.clarify` 命令用于交互式收集写作需求信息，确保生成的内容符合用户期望。

## 澄清流程

### 第一步：识别基本信息

从用户输入中提取：
1. **场景类型**：根据关键词自动检测
2. **初步需求**：用户描述的核心内容

### 第二步：逐项确认

需要确认的信息项：

#### 1. 收文对象
- 上级领导（需要更正式、谨慎）
- 同级部门（平等协商语气）
- 下级单位（指导性语气）
- 外部单位（礼貌、专业）
- 公众/社会（通俗易懂）

#### 2. 具体职务
- 如：处长、主任、总经理、局长等
- 影响称谓和敬语使用

#### 3. 场景类型
- report: 工作报告/总结
- letter: 请示/函件
- speech: 讲话/致辞
- briefing: 简报/汇报
- proposal: 方案/计划
- memo: 会议纪要

#### 4. 写作背景
- 事件背景
- 写作缘由
- 期望达成的目标

#### 5. 字数要求
- 500字以内：简短通知、便函
- 500-1000字：一般公文
- 1000-2000字：标准报告
- 2000-3000字：详细报告
- 3000字以上：综合性报告

#### 6. 语气风格
- formal: 正式严谨
- semi-formal: 平实稳重
- professional: 专业务实
- warm: 热情亲切

#### 7. 关键要点
- 必须包含的信息
- 重要数据
- 核心观点

#### 8. 参考资料
- 上级文件
- 政策法规
- 历史数据
- 会议精神

### 第三步：生成需求摘要

整理已收集的信息，形成完整的写作需求描述。

## 输出格式

```json
{
  "success": true,
  "message": "需求澄清完成",
  "writing_requirements": {
    "recipient_type": "上级领导",
    "recipient_title": "处长",
    "scenario": "report",
    "context": "...",
    "word_count": 2000,
    "audience": "处室领导",
    "purpose": "汇报工作",
    "tone": "formal",
    "key_points": "...",
    "references": "..."
  },
  "next_step": "使用 /writing.outline 生成大纲，或直接使用 /writing.generate 生成内容"
}
```

## 交互示例

**用户**：帮我写一份给领导的工作汇报

**助手**：
好的，我来帮您澄清写作需求：

1. **收文对象**：您提到是给领导，请问是哪个级别的领导？
   - 直接上级（如科长、处长）
   - 分管领导
   - 主要领导

2. **汇报内容**：是关于什么工作的汇报？
   - 日常工作进展
   - 专项工作
   - 年度/季度总结

3. **字数要求**：预期多长？
   
4. **重点内容**：有哪些必须体现的内容？

请补充以上信息，我将为您生成更精准的内容。
