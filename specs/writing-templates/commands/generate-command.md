# 内容生成命令模板

## 命令说明

`/writing.generate` 命令用于根据澄清的需求生成完整的公文内容。

## 生成流程

### 第一步：解析需求

从参数中提取：
1. 场景类型（自动检测或指定）
2. 收文对象
3. 写作背景
4. 字数要求
5. 语气风格
6. 关键要点

### 第二步：加载模板

根据场景类型加载对应模板：
- `report` → 工作报告模板
- `letter` → 请示函件模板
- `speech` → 讲话致辞模板
- `briefing` → 简报汇报模板
- `proposal` → 方案计划模板
- `memo` → 会议纪要模板

### 第三步：填充内容

1. **替换占位符**
   - `{{date}}` → 当前日期
   - `{{recipient}}` → 收文对象
   - `{{title}}` → 文书标题
   - 其他动态变量

2. **生成主体内容**
   - 按照模板结构组织内容
   - 融入关键要点
   - 控制字数

3. **应用格式规范**
   - 标题格式
   - 段落结构
   - 标点符号
   - 落款格式

### 第四步：质量检查

1. 字数是否符合要求
2. 结构是否完整
3. 语气是否一致
4. 格式是否规范

## 生成要求

### 语言规范
- 使用规范的书面语
- 避免口语化表达
- 使用公文常用词汇
- 表述准确、简洁

### 结构要求
- 层次分明
- 逻辑清晰
- 重点突出
- 首尾呼应

### 格式要求
- 标题居中
- 段落首行缩进两格
- 正确使用标点
- 规范的落款格式

## 输出格式

```json
{
  "success": true,
  "message": "内容生成完成",
  "generated_content": "（完整的公文内容）",
  "metadata": {
    "scenario": "report",
    "word_count": 1856,
    "structure": ["标题", "正文", "落款"]
  },
  "suggestions": [
    "建议根据实际情况补充具体数据",
    "落款日期请根据实际填写"
  ]
}
```

## 场景模板结构

### report - 工作报告

```
标题：关于XXX的工作报告

一、工作概述
二、主要工作及成效
    （一）...
    （二）...
三、存在问题
四、下一步计划
```

### letter - 请示函件

```
标题：关于XXX的请示

主送机关：XXX

正文：
（请示缘由）
（请示事项）
（结语）

落款
```

### speech - 讲话致辞

```
（称呼）

（开场）

一、（回顾/背景）
二、（主体内容）
三、（工作要求/展望）

（结语）
```

### proposal - 方案计划

```
标题：XXX实施方案

一、背景与目的
二、总体目标
三、主要措施
四、实施步骤
五、保障措施
```

### memo - 会议纪要

```
XXX会议纪要

时间：
地点：
主持人：
参会人员：

一、会议议题
二、讨论情况
三、会议决定
```
